Inputs y Outputs en Angular

Aprende la comunicación entre componentes usando inputs y outputs en Angular 16+

📡 Comunicación entre Componentes

Los inputs y outputs son la forma fundamental de comunicación entre componentes en Angular. Permiten pasar datos del componente padre al hijo (inputs) y eventos del hijo al padre (outputs).

🔄 Flujo de Comunicación

Componente Padre

Inputs → Outputs

↓ Inputs
Componente Hijo

Inputs ← Outputs

↑ Outputs

📥 Inputs

  • • Pasan datos del padre al hijo
  • • Se declaran con @Input()
  • • Binding: [property]="value"

📤 Outputs

  • • Emiten eventos del hijo al padre
  • • Se declaran con @Output()
  • • Event binding: (event)="handler()"

📥 Inputs (@Input)

Los inputs permiten que un componente hijo reciba datos del componente padre. Se declaran usando el decorador @Input().

Ejemplo de Input 
{{ inputExample }}

🔧 Configuración

  • • @Input() en la propiedad
  • • Binding en el template: [property]="value"
  • • Tipado TypeScript opcional

💡 Mejores Prácticas

  • ✓ Usar nombres descriptivos
  • ✓ Tipado explícito
  • ✓ Validación de datos
  • ✓ Valores por defecto

📤 Outputs (@Output)

Los outputs permiten que un componente hijo emita eventos al componente padre. Se declaran usando el decorador @Output() con EventEmitter.

Ejemplo de Output 
{{ outputExample }}

🔧 Configuración

  • • @Output() con EventEmitter
  • • Binding en el template: (event)="handler()"
  • • Método emit() para disparar eventos

💡 Mejores Prácticas

  • ✓ Nombres de eventos descriptivos
  • ✓ Datos estructurados en eventos
  • ✓ Validación antes de emitir
  • ✓ Documentación de eventos

🚀 Patrones Avanzados

🔄 Two-Way Binding

Combinación de input y output para sincronización bidireccional

[(ngModel)]="value"

📋 Content Projection

Pasar contenido HTML entre componentes

<ng-content></ng-content>

🎯 ViewChild

Acceso directo a componentes hijos

@ViewChild() childComponent

⚠️ Consideraciones Importantes

  • • Los inputs son readonly en el componente hijo
  • • Los outputs deben ser EventEmitter tipados
  • • Evitar mutar objetos pasados por input
  • • Usar OnPush change detection para mejor performance
