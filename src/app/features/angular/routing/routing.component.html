Angular Routing

Aprende a implementar navegación efectiva en Angular, desde conceptos básicos hasta técnicas avanzadas de routing y navegación programática.

¿Qué es Angular Routing?

El routing en Angular es el sistema que permite navegar entre diferentes vistas o componentes de tu aplicación sin recargar la página completa. Es fundamental para crear aplicaciones de página única (SPA) con múltiples vistas.

Características Principales

  • Navegación declarativa y programática
  • Parámetros de ruta y query strings
  • Guards de navegación
  • Lazy loading de módulos

Beneficios

  • Mejor experiencia de usuario
  • Navegación más rápida
  • URLs amigables y SEO
  • Código más organizado

Routing Básico

El routing básico en Angular se configura definiendo rutas que mapean URLs a componentes. Cada ruta especifica un path y el componente que se debe mostrar.

Configuración Básica de Rutas

{{ basicRoutingExample }}

Elementos Clave

  • RouterModule: Módulo que proporciona funcionalidad de routing
  • Routes: Array que define las rutas de la aplicación
  • RouterOutlet: Directiva que renderiza el componente de la ruta activa
  • routerLink: Directiva para navegación declarativa

Flujo de Navegación

  1. 1. Usuario hace clic en un enlace
  2. 2. Angular intercepta la navegación
  3. 3. Router busca la ruta correspondiente
  4. 4. Se renderiza el componente asociado
  5. 5. Se actualiza la URL en el navegador

Navegación en Angular

Angular proporciona dos formas principales de navegación: declarativa usando directivas y programática usando el servicio Router.

Navegación Declarativa

Parámetros de Ruta

Los parámetros de ruta permiten pasar datos entre componentes a través de la URL. Son fundamentales para crear rutas dinámicas y reutilizables.

Configuración de Parámetros

{{ paramsExample }}

Tipos de Parámetros

  • Path Parameters: /users/:id
  • Query Parameters: ?page=1&size=10
  • Matrix Parameters: /users;id=123
  • Optional Parameters: Parámetros opcionales

Acceso a Parámetros

  • ActivatedRoute.params: Parámetros de ruta
  • ActivatedRoute.queryParams: Query parameters
  • ActivatedRoute.data: Datos de resolvers
  • ActivatedRoute.fragment: Fragmentos de URL

Guards de Navegación

Los guards son interfaces que permiten controlar el acceso a rutas. Pueden prevenir la navegación, redirigir a otras rutas o cargar datos antes de activar una ruta.

Implementación de Guards

{{ guardsExample }}

Tipos de Guards

  • CanActivate: Controla acceso a rutas
  • CanDeactivate: Previene salida de rutas
  • CanLoad: Controla carga de módulos
  • Resolve: Precarga datos

Casos de Uso

  • Autenticación: Verificar login
  • Autorización: Verificar permisos
  • Validación: Verificar datos
  • Precarga: Cargar datos necesarios

Lazy Loading

El lazy loading permite cargar módulos y componentes solo cuando son necesarios, mejorando significativamente el rendimiento de la aplicación.

Beneficios

  • Tiempo de carga inicial: Más rápido
  • Uso de memoria: Optimizado
  • Ancho de banda: Reducido

Estrategias

  • Standalone: Componentes independientes
