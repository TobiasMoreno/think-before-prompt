🔧 Ejemplo: Debugging de Error Angular

Error común:

Error: Can't bind to 'ngModel' since it isn't a known property of 'input'

Solución con IA:

  • • Verificar importación de FormsModule
  • • Revisar declaraciones en el módulo
  • • Confirmar sintaxis de two-way binding
  • • Validar versión de Angular

⭐ Mejores Prácticas

✅ Qué Hacer

Revisar Código Generado

Siempre revisa y entiende el código que genera la IA antes de usarlo.

Usar Prompts Específicos

Sé específico en tus prompts para obtener resultados más precisos.

Mantener Control de Versiones

Usa Git para mantener un historial de cambios y poder revertir si es necesario.

Documentar Decisiones

Documenta por qué elegiste ciertas soluciones generadas por IA.

❌ Qué Evitar

Confiar Ciegamente

No uses código generado sin entenderlo o sin probarlo.

Prompts Vagos

Evita prompts genéricos que pueden generar código inadecuado.

Ignorar Seguridad

No ignores las implicaciones de seguridad del código generado.

Depender Exclusivamente

No dependas exclusivamente de la IA; mantén tus habilidades de programación.
